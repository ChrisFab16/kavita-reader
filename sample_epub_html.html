Chapter 1: The Beginning

It started with a simple idea: Emily should be able to read her books anywhere, on any device. The existing Kavita Android app was clunky, and there was no iOS option at all.

On November 29, 2025, Nicholas sat down with Claude (Anthropic's AI assistant) with minimal coding experience beyond basic shell scripts. The goal? Build a modern, cross-platform mobile reader app as a Christmas gift.

Starting Point:
  • Experience: Basic shell scripting, bash
  • Goal: Professional iOS/Android app
  • Timeline: Christmas 2025
  • Tools: React Native, TypeScript, Expo

Chapter 2: The Tech Stack

Choosing the Right Tools

After research, we settled on:

  • React Native + Expo: Write once, deploy to iOS and Android
  • TypeScript: Catch errors before they happen
  • React Native Paper: Beautiful Material Design 3 components
  • Zustand: Simple, powerful state management
  • Axios: Reliable API communication

The Kavita API

We dove deep into Kavita's REST API, studying the OpenAPI specification and understanding how to:

  • Authenticate with JWT tokens
  • Fetch libraries, series, and chapters
  • Handle PDF extraction and image caching
  • Track reading progress
  • Support multiple server connections

Chapter 4: Challenges & Solutions

The Emoji Crisis

Book icons appeared as weird symbols instead of emojis. Solution? Replace them all with Material Design icons. Problem solved!

The Loading Loop

PDFs would load... and load... and load. The issue? We forgot to set loading = false after caching. One line of code. Classic.

The 400 Bad Request Mystery

Pages wouldn't load. The error? We were sending page=1 when Kavita uses 0-based indexing. It wanted page=0. Off-by-one errors strike again!

🎯 Lesson Learned: Read the API docs carefully. Then read them again. Then check the actual API calls in the browser network tab.

Dark Mode Inconsistency

The theme would work on some screens but not others. The fix? Create a centralized theme store using Zustand and ensure every screen subscribes to it.
