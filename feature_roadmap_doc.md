## 🎯 Confirmed for Next Release (v1.1.0)

**Target:** Q1 2025 (January-March)

### Core Features
- [ ] **App Update Checker** - Check for new versions of KavitaReader
  - In-app notification when update available
  - Link to Play Store for update
  - "What's New" changelog display

- [ ] **Library Update Notifications** - Get notified when new books are added
  - Background check for new content on server
  - Push notification when library updates
  - Show count of new books added

- [ ] **Notification Settings Panel**
  - Toggle app update notifications on/off
  - Toggle library update notifications on/off
  - Set check frequency (hourly, daily, weekly)
  - Quiet hours settings

### Additional v1.1.0 Features
- [ ] **Reading Goals** - Set daily/weekly/monthly reading targets
  - Track pages or time spent reading
  - Progress indicators
  - Achievement notifications

- [ ] **Audiobook Support** - Play audiobooks from Kavita
  - Basic playback controls
  - Background playback
  - Resume where you left off
  - Sleep timer integration

## 🤔 Under Consideration

Features being evaluated for future versions:

### High Priority
- **Collections/Custom Lists** - Create personal reading lists
- **Reading Streaks** - Track consecutive days reading
- **Offline Downloads Manager** - Choose books to download
- **Home Screen Widget** - Show currently reading book
- **Sleep Timer** - Auto-close after X minutes
- **Blue Light Filter** - Warmer colors for night reading

### Medium Priority
- **Reading Statistics Dashboard** - Detailed reading analytics
- **Quick Series Access** - Long-press to jump to series
- **Recently Read Shelf** - Last 10 books accessed
- **Reading Heatmap** - Calendar view of activity
- **Double-Page View** - For tablets/landscape
- **Font Picker** - Multiple reading font options

### Low Priority / Future
- **Text-to-Speech** - Read books aloud
- **Social Features** - Share with friends/family
- **Book Club Mode** - Private groups
- **Custom Themes** - User-created color schemes
- **iOS Version** - iPhone and iPad support

## 🎯 Feature Prioritization Criteria

When deciding what to build next, we consider:

1. **User Impact** - How many users benefit?
2. **Alignment with Vision** - Does it fit KavitaReader's purpose?
3. **Technical Feasibility** - Can we build it reliably?
4. **Maintenance Burden** - How much ongoing work required?
5. **Emily's Preference** - Would Emily find this useful? 😊
6. **Privacy First** - Does it respect user privacy?

## 💭 Notes & Philosophy

**Design Principles:**
- Privacy first, always
- Reading experience over features
- Simple by default, powerful when needed
- Emily-approved features get priority 😊
- Community input valued and considered

**What We Won't Build:**
- Features that compromise privacy
- Bloated UI that distracts from reading
- Ads or data collection
- Paywalled features (app stays free)
- Anything that requires cloud infrastructure we control
